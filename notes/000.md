This is an elite setup. Using an **M1 Max** with **macOS Tahoe 26.4** gives you the hardware acceleration needed for fast local compilation, while **RustRover 2026.1** now features native **ACP (Agent Client Protocol)** support, allowing you to plug agents directly into the IDE.

Here is your step-by-step installation checklist to build the "Toolbox" environment.

---

### Phase 1: The Foundation (Host Machine)
Before jumping into Docker, we need the workspace structure and the AI "Instructions" ready.

1.  **Create the Root Workspace:**
    Open your terminal and run:
    ```bash
    mkdir ~/rust-toolbox && cd ~/rust-toolbox
    mkdir tools
    touch Cargo.toml AGENTS.md
    ```
2.  **Initialize the Virtual Manifest:**
    Open the root `Cargo.toml` in RustRover and paste:
    ```toml
    [workspace]
    resolver = "2"
    members = ["tools/*"]

    [workspace.dependencies]
    clap = { version = "4.5", features = ["derive"] }
    anyhow = "1.0"
    tokio = { version = "1.38", features = ["full"] }
    ```
3.  **Create the Master AI Instructions:**
    In `AGENTS.md`, add your preferences so the agent doesn't guess:
    ```markdown
    # Rust CLI Agent Rules
    - Target Architecture: aarch64-apple-darwin (M1 Max)
    - Code Style: Functional, minimal use of 'unsafe'.
    - Error Handling: Use 'anyhow' for CLI apps.
    - Workflow: Run 'cargo check' after every file change.
    ```

---

### Phase 2: The Sandbox (Docker v4.67.0)
Since you are on an M1 Max, we will use an **ARM64-native** Rust image to ensure the agent's compile speeds stay blazing fast.

1.  **Configure Docker Desktop:**
    * Ensure **VirtioFS** is enabled in *Settings > General* (this is crucial for macOS Tahoe file-sharing performance with Rust's heavy `target` folders).
    * Ensure **Rosetta 2** is enabled in *Settings > Features* (just in case an agent pulls an x86-only crate).
2.  **Create the Development Container:**
    Create a folder `.devcontainer` in your root and add `devcontainer.json`:
    ```json
    {
      "name": "Rust Toolbox",
      "image": "mcr.microsoft.com/devcontainers/rust:1-1-bookworm",
      "features": {
        "ghcr.io/devcontainers/features/common-utils:1": {}
      },
      "customizations": {
        "jetbrains": {
          "plugins": ["org.rust-lang.rustrover"]
        }
      },
      "remoteUser": "vscode"
    }
    ```

---

### Phase 3: IDE & Agent Integration
RustRover 2026.1 has a "Remote Development" mode that is perfect for this.

1.  **Connect RustRover to Docker:**
    * In RustRover, go to **File > Remote Development**.
    * Select **Docker**.
    * Point it to your `~/rust-toolbox` folder. RustRover will now build the container and start a "Backend" process inside it.
2.  **Enable the Agent via ACP:**
    * Go to **Settings > Tools > AI Agents**.
    * Since you are in 2026.1, you can select **"Enable Agent Client Protocol (ACP)"**.
    * If you use **Claude Code** or **Aider**, you can now toggle the "Terminal Agent" to have write-access to your workspace.

---

### Phase 4: Daily Workflow (The First Tool)
To test the environment, give your agent this command in the RustRover AI Chat:

> "Create a new CLI tool in the workspace called `fetch-it`. It should take a URL as an argument and print the status code. Use the workspace dependencies."

**Checklist for Success:**
* [ ] Does `tools/fetch-it/Cargo.toml` exist?
* [ ] Does the root `target/` folder show architecture `aarch64`?
* [ ] Can you run `cargo run -p fetch-it -- https://google.com` from the IDE terminal?

**One Final Tip for M1 Max Users:**
Because the M1 Max has so many performance cores, tell your agent to use `cargo-nextest`. It’s much faster at parallelizing tests than the standard `cargo test`, and your machine can easily handle 10+ concurrent test threads.
